# ToolSync Development Plan

## 목적
로컬 중심 ToolSync 확장을 안정적으로 반복 개발하고, 릴리즈 품질을 유지한다.

## 기본 루프
1. 요구사항 반영
2. 최소 구현
3. 로컬 검증(compile/package/install)
4. 문서/체인지로그 갱신
5. 커밋/태그/릴리즈

## 필수 중간 단계: 클린업
모든 개발 작업은 중간에 반드시 **클린업** 단계를 1회 이상 수행한다.

클린업 체크리스트:
1. 불필요 산출물/임시파일을 `archive/`로 이동
2. `archive/`는 git 추적 제외 상태 유지 (`.gitignore`)
3. 무거운 동작 점검
   - 불필요한 반복 연산 제거
   - 불필요한 파일 I/O 감소
   - 필요 없는 전체 스캔/재계산 제거
4. 용량 점검
   - 번들/산출물에 불필요 파일 포함 여부 확인
5. 동작 영향 점검
   - 기능 유지 + 성능 개선 확인

## 릴리즈 기준
1. `npm run compile` 성공
2. `npm run package` 성공
3. VSIX 설치 성공
4. README 설치/운영 안내 최신화
